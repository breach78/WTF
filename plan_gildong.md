## 2. 이번 재작업의 핵심 판단

기존 결과물의 문제는 수량이 아니라 계층의 역할 분리가 무너졌다는 점이다.

- `4막`은 표지판처럼 제목만 있었고, 막 자체의 서사 요약 기능이 약했다.
- `개요`는 영화 전체를 한 번에 조망하는 핵심 구조 비트가 아니라, 하위 요약을 기계적으로 잘라 붙인 느낌에 가까웠다.
- `시놉시스`는 개요보다 서사적으로 더 풀어져야 하는데, 실제로는 개요와 거의 같은 밀도로 붙어 있었다.
- `시퀀스`는 시놉시스의 하위 사건 단위여야 하는데, 실제로는 시놉시스와 거의 1:1로 대응해 차이가 흐려졌다.
- `트리트먼트`는 수정 작업에 바로 사용할 수 있을 정도로 세밀해야 하는데, 아직은 한 카드 안에 너무 많은 사건이 묶여 있었다.

따라서 이번에는 `몇 개 이상` 같은 기계적 최소 규칙으로 구조를 강제하지 않는다.
대신, 각 계층이 아래 계층으로 내려갈수록 반드시 더 구체적이고 더 풍성해야 한다는 질적 기준으로 재설계한다.

## 2-1. 원고 고유 특성 반영

이번 재설계는 일반적인 플롯 문서를 만드는 작업이 아니다.
기준 원고는 다음 성질 때문에, 요약 프롬프트가 조금만 평면적으로 가도 품질이 바로 무너진다.

- 장면 전환이 빠르고 `현재 / 과거 / 상상 / 미래 / 회상`이 자주 왕복한다.
- 인물 내면보다 `행동`, `공간 점유`, `군중의 반응`, `폭력의 확산`으로 의미를 만든다.
- 대사는 정보 전달보다 `계급 감각`, `조롱`, `선동`, `자기방어`의 기능이 강하다.
- `홍길동`은 개인 이름이 아니라 `소문`, `가면`, `정체성`, `집단적 이름`으로 확장된다.
- 군중은 배경이 아니라 사건 방향을 실제로 바꾸는 주체다.
- 결말의 핵심은 승패보다 `홍길동이라는 이름이 한 사람에서 다수에게 이동하는 과정`이다.

따라서 이번 작업의 품질 기준은 다음 네 축을 반드시 포함한다.

- `계층 분리`
- `시간축 분리`
- `군중성 보존`
- `상징 축 보존`

## 3. 목표 구조

- `4막`: 4개
- `개요`: 10~11개
- `시놉시스`: 개요보다 자연스럽게 더 많아지는 구조
- `시퀀스`: 시놉시스보다 자연스럽게 더 많아지는 구조
- `트리트먼트`: 시퀀스보다 훨씬 많고, 가능하면 씬에 가까운 수정 단위
- `원문`: `116씬` 그대로 유지

중요:

- 개수는 목표일 뿐, 강제 규칙이 아니다.
- 상위 계층을 제대로 분해하면 하위 계층이 더 많아지는 것은 결과이지, 형식적 의무가 아니다.
- 판단 기준은 숫자가 아니라 `역할 차이`와 `밀도 차이`다.

## 4. 계층별 정의

### 4-1. 4막

`시작 / 중간1 / 중간2 / 끝`은 단순한 표제어가 아니라, 각 막 전체를 한 문단으로 요약하는 상위 서사 단위다.

각 4막 카드에는 다음이 들어가야 한다.

- 그 막에서 무슨 일이 벌어지는지
- 주인공의 상태가 어떻게 바뀌는지
- 드라마의 목표와 적이 어떻게 이동하는지
- 다음 막으로 넘어가게 만드는 전환점이 무엇인지

즉 `1막의 내용`, `2막 전반의 내용`, `3막의 내용`, `4막의 내용`이 각각 산문으로 요약되어야 한다.

### 4-2. 개요

`개요`는 영화 전체를 기억하게 만드는 핵심 구조 비트다.

- 이 개요만 순서대로 읽어도 영화 전체가 머릿속에 큰 흐름으로 그려져야 한다.
- 너무 잘게 쪼개면 안 되고, 너무 뭉뚱그려도 안 된다.
- 사건의 전환점, 목표 변화, 주인공의 결정, 영화적 방향 전환 같은 핵심 비트 위주여야 한다.

개요는 하위 요약을 기계적으로 묶어 만드는 것이 아니라, 시나리오 전체를 먼저 종합해서 다시 뽑는다.

### 4-3. 시놉시스

`시놉시스`는 각 개요를 더 서사적으로 풀어쓴 하위 단위다.

- 개요가 구조적 핵심이라면, 시놉시스는 그 핵심이 어떤 사건 흐름으로 구현되는지 보여줘야 한다.
- 개요보다 문장 수와 정보량이 분명히 더 많아야 한다.
- 개요 하나 아래에 들어온 시놉시스들을 합치면, 그 개요의 내용이 구체적인 이야기로 확장되어야 한다.

### 4-4. 시퀀스

`시퀀스`는 시놉시스를 이루는 실제 진행 단위다.

- 보통 하나의 목표, 작전, 상황, 감정 흐름, 장소군을 중심으로 작동한다.
- 하나의 시퀀스는 작은 기승전결이나 작은 완결성을 가져야 한다.
- 시놉시스를 더 작고 더 움직이는 사건 단위로 분해한 결과물이어야 한다.

### 4-5. 트리트먼트

`트리트먼트`는 실제 수정 작업에 바로 사용할 수 있는 세밀한 산문 단위다.

- 시퀀스를 더 잘게 풀어 쓴다.
- 사건, 행동, 갈등, 서브텍스트, 전환을 가능한 한 구체적으로 적는다.
- 이상적으로는 `씬에 거의 근접한 수정 단위`가 되어야 한다.

원칙:

- 기본은 `트리트먼트 1개 = 원문 1씬`에 가깝게 가져간다.
- 다만 정말로 분리할 의미가 없는 아주 짧은 연결 씬만 2~3씬 정도 묶는다.
- 어떤 트리트먼트가 2씬 이상을 묶으면, 왜 묶었는지 이유를 적는다.

### 4-6. 원문

- 말단 카드에는 원문만 둔다.
- 원문은 기준 파일에서 직접 슬라이스한다.
- `1자`, `공백`, `줄바꿈`, `빈 줄`까지 그대로 유지한다.
- 원문 카드는 설명, 번호, 요약 문장 없이 원문만 들어간다.

## 5. 설계 원칙

### 5-1. 상위에서 하위로 갈수록 풍성해질 것

각 계층은 바로 아래 계층보다 반드시 추상적이어야 하고, 바로 위 계층보다 반드시 구체적이어야 한다.

즉:

- `4막`은 가장 압축된 상위 서사
- `개요`는 그보다 구체적
- `시놉시스`는 더 풀어진 이야기
- `시퀀스`는 실제 사건 전개 단위
- `트리트먼트`는 거의 수정 가능한 현장 단위
- `원문`은 최종 세부 레벨

### 5-2. 1:1 대응이 생기면 경계할 것

계층 간 수량이 우연히 비슷할 수는 있지만, 역할까지 비슷해지면 실패다.

검토 포인트:

- `개요`와 `시놉시스`가 사실상 같은 말을 하고 있지 않은가
- `시놉시스`와 `시퀀스`가 사실상 같은 단위로 겹치지 않는가
- `시퀀스`와 `트리트먼트` 사이에 실제 밀도 차이가 있는가

### 5-3. 상위 구조는 하위에서 역산하지 않는다

이번에는 `트리트먼트 -> 시퀀스 -> 시놉시스 -> 개요` 식으로 기계적으로 묶어 올리지 않는다.

대신 순서를 이렇게 바꾼다.

1. 시나리오 전체를 다시 읽고 `4막`과 `개요`를 먼저 잡는다.
2. 그다음 각 개요를 `시놉시스`로 확장한다.
3. 각 시놉시스를 `시퀀스`로 분해한다.
4. 각 시퀀스를 `트리트먼트`로 세분한다.
5. 마지막으로 각 트리트먼트에 원문을 매핑한다.

즉, 상위 구조를 먼저 제대로 잡고 아래로 내려가는 방식으로 바꾼다.

### 5-4. 요약 품질 향상 프로토콜

이번 플랜에서는 각 단계별 요약을 즉석에서 감으로 쓰지 않는다.
사전에 연구한 프롬프트와 자가검수 루프를 붙여서, 각 단계의 품질을 통제한다.

핵심 원칙:

- 각 계층은 바로 위 계층보다 더 풍성해야 한다.
- 시간 점프는 절대 평평하게 합쳐 쓰지 않는다.
- 군중을 배경으로 축소하지 않는다.
- `홍길동`의 상징 확장을 끝까지 유지한다.
- 장면 간 `원인 -> 결과`를 반드시 드러낸다.

실패 신호:

- 현재/과거/상상 장면이 하나의 직선 줄거리로 뭉개진다.
- 왕, 허균, 길, 개똥, 이이첨이 단순 선악으로 납작해진다.
- 군중이 빠지고 영웅 서사만 남는다.
- 개요와 시놉시스, 시놉시스와 시퀀스가 같은 밀도로 반복된다.
- 결말이 `길의 희생`만 남고 `이름의 집단화`가 사라진다.

## 5-5. 단계별 생성 프롬프트

아래 프롬프트는 실제 작업 시 기본 생성 프롬프트로 사용한다.

```text
당신은 시나리오 구조 분석가이자 스크립트 닥터다.

기준 원고를 읽고, 이번 단계의 목표 계층만 작성하라.
단, 아래 원칙을 반드시 지켜라.

핵심 원칙:
1. 이 작품은 군중 서사, 정치 스릴러, 민중 우화가 겹친 원고다. 개인 사건만 요약하지 말고, 군중의 움직임과 권력 변화를 포함하라.
2. 현재 / 과거 / 상상 / 미래 / 회상 장면을 반드시 구분하라. 시간 점프를 하나의 직선 줄거리로 합쳐 쓰지 마라.
3. 각 단위는 "무슨 일이 벌어졌는가"보다 "왜 이 사건이 다음 사건을 불렀는가"를 보여줘야 한다.
4. 허균, 길, 개똥, 왕, 이이첨을 평면적으로 요약하지 말고, 각각이 대표하는 질서와 긴장을 밝혀라.
5. 홍길동은 한 개인이 아니라 이름, 소문, 가면, 집단 정체성으로 확장된다. 이 축을 계속 유지하라.
6. 중요한 전환마다 전환의 이유를 한 줄로 적어라.
7. 원문 대사는 길게 인용하지 말고, 필요한 경우에만 짧게 의미만 요약하라.
8. 불확실한 부분은 단정하지 말고 "추정" 또는 "의도상"이라고 표시하라.

이번 단계의 계층 역할:
- 4막: 되돌릴 수 없게 바뀐 큰 구조 변화
- 개요: 영화 전체를 기억하게 만드는 핵심 구조 비트
- 시놉시스: 개요를 더 서사적으로 풀어쓴 하위 이야기
- 시퀀스: 같은 목표/상황/시간대의 사건 단위
- 트리트먼트: 화면 전개와 행동을 따라가는 수정용 산문 단위

품질 기준:
- 시간축이 맞아야 한다
- 인과가 끊기지 않아야 한다
- 군중과 권력의 관계가 보여야 한다
- 결말의 집단적 의미가 살아 있어야 한다
- 현재 단계가 바로 위 계층보다 분명히 더 풍성해야 한다
```

## 6. 실행 단계

### Phase A. 전체 재독과 4막 재설정

목표:

- 영화 전체를 다시 파악하고 `4막`을 새로 쓴다.

작업:

- 현재 카드 구조는 참고만 하고, 기준 원고를 다시 읽는다.
- `시작 / 중간1 / 중간2 / 끝` 각각에 대해 막 전체의 산문 요약을 새로 쓴다.
- 각 막마다 전환점과 기능을 분명히 잡는다.

산출물:

- `4막 요약표`

승인 게이트:

- 각 막이 제목이 아니라 실제 요약으로 기능하는지 확인받는다.
- 각 막 요약이 시간축, 군중성, 상징 축을 놓치지 않았는지 자가검수한다.

### Phase B. 개요 10~11개 재설계

목표:

- 영화 전체의 핵심 구조 비트 `10~11개`를 새로 뽑는다.

작업:

- 사건의 발단, 방향 전환, 목표 이동, 몰락, 선언, 역공, 붕괴, 희생, 집단적 귀환 같은 구조 비트를 중심으로 개요를 다시 잡는다.
- 개요만 읽어도 영화가 큰 흐름으로 재생되어야 한다.

산출물:

- `개요 구조표`

승인 게이트:

- 개요만 읽고 영화 전체 순서가 머릿속에 그려지는지 확인받는다.
- 개요가 하위 사건을 기계적으로 묶은 문장이 아니라, 영화 전체를 다시 종합한 구조 비트인지 검수한다.

### Phase C. 시놉시스 설계

목표:

- 각 개요를 더 구체적인 이야기 덩어리들로 확장한다.

작업:

- 개요마다 어떤 이야기 단위들로 나뉘는지 다시 판단한다.
- 각 시놉시스는 그 개요보다 분명히 풍성해야 한다.
- 개요 안에 들어 있는 감정선과 사건 흐름이 어떻게 실제 서사로 전개되는지 보여준다.

산출물:

- `개요 -> 시놉시스 매핑표`

승인 게이트:

- 각 개요 아래의 시놉시스가 실제로 더 서사적이고 더 풍성한지 검토받는다.
- 시놉시스가 개요의 반복문이 되지 않았는지, 시간 점프와 인과를 더 잘 드러내는지 검수한다.

### Phase D. 시퀀스 설계

목표:

- 시놉시스를 실제 사건 전개 단위로 분해한다.

작업:

- 각 시놉시스를 구성하는 목표, 상황, 장소군, 작전, 감정 흐름에 따라 시퀀스를 나눈다.
- 시퀀스는 `무엇이 진행되고 어떤 작은 완결에 도달하는지`가 보여야 한다.

산출물:

- `시놉시스 -> 시퀀스 매핑표`

승인 게이트:

- 시퀀스가 시놉시스의 축소판이 아니라, 더 작은 사건 단위로 기능하는지 확인받는다.
- 시퀀스마다 목표, 상황, 전환점이 분명한지 검수한다.

### Phase E. 트리트먼트 세분화

목표:

- 시퀀스를 수정 가능한 밀도로 잘게 푼다.

작업:

- 시퀀스를 가능한 한 씬에 가까운 서술 단위로 쪼갠다.
- 각 트리트먼트는 사건, 행동, 갈등, 변화가 분명해야 한다.
- 여러 씬을 묶을 경우는 연결 씬 여부를 보고 판단하고, 묶은 이유를 적는다.

산출물:

- `시퀀스 -> 트리트먼트 매핑표`

승인 게이트:

- 트리트먼트가 실제 수정 단위로 쓸 수 있을 정도로 충분히 세밀한지 확인받는다.
- 여러 씬이 묶였을 경우 왜 묶였는지 이유를 남기고 검수한다.

### Phase F. 원문 재매핑

목표:

- 최종 트리트먼트 말단에 원문 씬을 정확히 붙인다.

작업:

- 기준 원고의 `EXT./INT.` 헤딩 기준 `116씬`을 다시 잠근다.
- 각 트리트먼트 하위에 해당 원문 씬 카드를 붙인다.
- 원문은 슬라이스로만 넣고, 저장 후 다시 읽어서 동일성 검산을 한다.

산출물:

- `트리트먼트 -> 원문 매핑표`
- `원문 동일성 검산표`

### Phase G. 앱 반영

목표:

- 승인된 구조만 앱 트리에 반영한다.

작업:

- 먼저 `샘플 브랜치 1개`만 시험 반영한다.
- 구조와 밀도가 맞다고 확인되면 전체를 확장한다.
- 기존 잘못된 구조는 필요 시 `archived` 처리한다.

산출물:

- 최종 앱 카드 트리

## 7. 검수 기준

이번 검수는 숫자보다 계층의 기능 차이에 집중한다.

체크리스트:

- `4막`은 막 전체를 요약하는가
- `개요`는 영화의 핵심 구조 비트만 남기는가
- `시놉시스`는 개요를 더 풍성하게 푸는가
- `시퀀스`는 시놉시스를 실제 사건 단위로 쪼개는가
- `트리트먼트`는 수정 가능한 세밀한 단위인가
- `원문`은 116씬 그대로 보존되는가
- `시간축`이 혼동되지 않는가
- `군중성`이 빠지지 않는가
- `홍길동`의 상징 확장이 결말까지 유지되는가

실패 신호:

- 상하위 계층이 같은 말을 반복한다
- 카드 수만 달라졌고 실제 밀도 차이는 없다
- 한 카드에 사건이 너무 많이 묶여 수정 단위로 쓰기 어렵다
- 상위 구조가 시나리오 전체를 종합하지 못한다

## 8. 원문 무결성 규칙

- 기준 원고 고정값은 유지한다.
- 총 씬 수는 `116`
- 씬 구분 기준은 오직 `EXT.` / `INT.`
- 원문은 카드 작성 때 사람이 재입력하지 않는다
- 원문 카드는 기준 원고 슬라이스만 사용한다
- 최종 저장 후 다시 읽어서 해시와 바이트를 검산한다

## 9. 이번 플랜의 결론

이번 재작업은 `구조를 더 많이 만드는 작업`이 아니라 `계층마다 다른 역할을 회복하는 작업`이다.

또한 이번에는 프롬프트 자체를 품질 통제 장치로 사용한다.
즉, `좋은 구조를 잡고 -> 그 구조에 맞는 생성 프롬프트를 쓰고 -> 자가검수 프롬프트로 다시 걸러내는 방식`으로 단계별 품질을 끌어올린다.

따라서 다음 실제 실행 순서는 이렇게 간다.

1. `4막 요약` 새로 작성
2. `개요 10~11개` 새로 작성
3. 사용자 확인
4. `시놉시스` 확장
5. 사용자 확인
6. `시퀀스` 분해
7. 사용자 확인
8. `트리트먼트` 세분화
9. 사용자 확인
10. 앱 샘플 반영
11. 전체 반영
